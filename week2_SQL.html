SQL

Objectives

  • Be able to retrieve data from database using SELECT
    能够检索数据从数据库, SELECT
  • Use compound WHERE conditions.
    使用复合条件, WHERE。
  • Sort query results using ORDER BY.
    对查询结果进行排序, ORDER BY
  • Use aggregate functions.
    使用聚合函数。
  • Group data using GROUP BY and HAVING.
    组合数据，GROUP BY & HAVING
  • Use subqueries.
    子查询
  • Join tables together.
    将表连接在一起
  • Perform set operations ,UNION, INTERSECT, EXCEPT.
    执行集合操作, UNION, INTERSECT, EXCEPT
  • Be able to update database using INSERT, UPDATE, and DELETE.
    更新数据库, INSERT, UPDATE, DELETE

Introduction to SQL

  • SQL is a transform-oriented language with 2 major components.

DDL & DML

  • Data definition language [DDL]:Create table, ALTER table, Drop table
  • Data manipulation language [DML]:Insert, Delete, Update, Select.

条件语句集 Condition

  • Compound Comparison Search Condition 复合比较搜索条件
WHERE column_name = “A” OR column_name = “B”;
WHERE column_name1 = “A” AND column_name2 = “A”;
  • Comparison Search Condition 比较搜索条件
WHERE column_name > 10000;
WHERE column_name = 10000;
  • Range Search Condition 范围搜索条件
WHERE column_name BETWEEN 20000 AND 30000;
  • Pattern Matching 正则匹配
WHERE column_name LIKE '%Glasgow%';
WHERE column_name LIKE '_英_';

数据库查询模糊匹配

  • NULL Search Condition NULL搜索条件
WHERE column_name IS NULL;

DDL命令集

指令 语句
创建数据库 CREATE DATABASE database_name;
展示所有数据库 SHOW DATABASES;

  • 建立新的表 CREATE
CREATE TABLE table_name (
    column_name data_type column_constraint,       # 列级约束性条件
    No char(2) PRIMARY KEY，                       # eg1: 主键约束
    BranchNo char(4) NOT NULL,                     # eg2：允许或不允许出现NULL
    street varchar(45) DEFAULT NULL,               # eg3：默认为NULL
    value integer(100) UNIQUE,                     # eg4：唯一性约束，值不能重复
    table_constraint,                              # 表级约束条件
    PRIMARY KEY (column_name),                     # eg1：主键约束
    CHECK (column_name Condition),                 # eg2：条件约束
    UNIQUE (column_name)                           # eg3：唯一性约束
    )ENGINE = InnoDB DEFAULT CHARSET = utf8;       # table_constraint

ALTER TABLE table_name DROP column_name;  # 删除属性

Aggregates functions 聚合函数

  1. COUNT: returns number of values in specified column.
  2. SUM: returns sum of values in specified column.
  3. AVG: returns average of values in specified column.
  4. MIN: returns smallest value in specified column.
  5. MAX: returns largest value in specified column.
聚合函数使用条件
  1. 每个操作的对象都是一个属性对应的一列，返回值也是一个值
  2. 关于值的类型是否为数字
    • COUNT、MIN、MAX的使用属性对象可以是数字或非数字类型的
    • SUM、AVG的适用对象必须保证是数字类型的。
  3. 关于值为null的影响
    • 除了COUNT函数，其他函数在检索某个列的所有值时会自动排除null值对应的项。
    • COUNT函数会得到某个表中的所有项的数量，不管是不是null
  4. 关于使用DISTINCT产生的效果
    • 复习一下，DISTINCT的作用是在处理数据时筛选掉值重复的行。
    • 对MIN、MAX没有效果
    • 对SUM、AVG、COUNT会产生效果
  5. 使用聚合函数必须在SELECT语句中使用
  6. SELECT语句不能同时选择一列和一个聚合函数组成一个新表(没有逻辑关系)，不过我们可以利用GROUP BY语句通过某一个属性值将行分类后依次执行聚合函数。
聚合函数的基本使用
  • COUNT（*）：统计所有行数量
SELECT COUNT(*) AS myCount
FROM column_name
(WHERE condition)

Result：

myCount
5
  • COUNT（DISTINCT columnname）：统计某一列值不重复的数量
SELECT COUNT(DISTINCT column_name) AS myCount
FROM column_name
(WHERE condition)

Result：

myCount
2
  • 组合使用聚合函数
SELECT COUNT(DISTINCT column_name) AS myCount
MIN(column_name) AS myMin
AVG(column_name) AS myAvg
FROM column_name
(WHERE condition)

Result：

myCount myMin myAvg
2 12000.00 15000.00
聚合函数的分类语句- Group BY

  • 如果SELECT语句的对象属性中只包含聚合函数，他的聚合对象就是默认的全部行。

  • 如果我们想在聚合函数统计之前先将数据分类，分别得到每类的聚合结果，我们就需要用到该语句。

  • 如果条件语句(WHERE)与GROUP BY语句同时存在，一定要保证条件语句在前,因为我们想要得到的数据是通过条件语句筛选后再通过聚合函数计算得到的结果。

  • GROUP BY 的使用

SELECT COUNT(column_name) AS myCount
AVG(column_name) AS myAvg
FROM column_name
(WHERE condition)
GROUP BY group_column_name；

Result:

branchNo myCount myAvg
B001 5 14000.00
B003 3 23000.00
B005 2 18000.00

GROUP BY的例子

聚合函数分类的筛选语句- HAVING
  • HAVING和WHERE的功能是很相似的，不过HAVING语句是纯为GROUP BY语句服务的。
  • HAVING的条件筛选对象是GROUP BY之后分出来的组
  • WHERE的条件筛选对象是表中的每一行
  • HAVING语句的语法是和条件语句的语法相同的，也可以叠加使用，按先后顺序进行筛选。
  • HAVING 的使用
SELECT COUNT(column_name) AS myCount
AVG(column_name) AS myAvg
FROM column_name
(WHERE condition)
GROUP BY group_column_name
HAVING COUNT(column_name) > 2;

Result:

branchNo myCount myAvg
B001 5 14000.00
B003 3 23000.00

Subqueries 子查询

子查询示例

Subquery with Equality 等式子查询
SELECT column_name
FROM column_name
WHERE spec_column_name =
(SELECT column_name
FROM table_name
WHERE column_name = 'Name')
  • 括号内为子查询内容，它会将查询结果作为值赋给外部语句
  • 优先执行子查询语句，后执行外部语句
  • 等式子查询只能解决子查询语句返回单行数据的情况，多行数据需要用到IN语句
Subquery with Aggregate 聚合式子查询
SELECT column_name
FROM column_name
WHERE spec_column_name =
(SELECT AVG(column_name)
FROM table_name)
  • 这里注意聚合函数的语法，一个聚合函数没有返回值，它只能配合SELECT语句得到一张表，所以切忌这样使用。
WHERE salary > AVG(salary)
  • 子查询式SELECT得到的筛选列表必须包含至少一列属性，除非子查询语句使用EXISTS(返回值是bool值)
Nested subquery 嵌套子查询
  1. IN 用法
SELECT column_name
FROM column_name
WHERE spec_column_name IN
(SELECT column_name
FROM table_name
WHERE condition)
  • 这里"IN"用来解决原本"="为解决的问题，即如果子查询的结果不为一个单行的表（只有一条数据）的情况
  1. ANY/SOME 用法
SELECT column_name
FROM column_name
WHERE spec_column_name > SOME(SELECT column_name
FROM table_name
WHERE condition)
  • ANY/SOME 含义相同
  • 两者起到的效果是将子查询返回的一张多行的表变成单行数据
  • 其创造的条件关系是只要满足子查询结果中任意一行数据即成立。
  1. ALL 用法
SELECT column_name
FROM column_name
WHERE spec_column_name > ALL(SELECT column_name
FROM table_name
WHERE condition)
  • 与ANY/SOME效果相同
  • 其创造的条件关系是要求满足子查询结果中每一行数据即成立。

Multi-Table Queries 多表查询

  • 在多表查询时，SELECT语句与单表查询相同，挑选你想得到的属性，逗号分隔。
  • 在单表查询时，FROM语句只能声明一张表，但在多表查询时可以将多张表作为筛选范围，逗号分割。
  • 在多表查询时，在表示某个表中属性时，若不涉及不同表中属性名相同，则属性的表归属可以省略，否则需要在前面声明其对应的表，用点连接
  • 在多表查询时，FROM语句中的表如果来自不同数据库，则需要声明数据库，数据库与表用空格连接
SELECT c.clientNo, fName, lName
FROM Client c,Viewing v
WHERE c.clientNo = v.clientNo
多表查询计算法则
  • 笛卡尔积法则：两个集合X和Y的笛卡尔积（Cartesian product），又称直积，表示为X × Y，第一个对象是X的成员而第二个对象是Y的所有可能有序对的其中一个成员。
  • 两张表通过计算笛卡尔积融合成一张表，再筛选出符合条件语句的行，生成一个新表。挑选SELECT语句中的属性进行显示。
  • DISTINCT 可以筛选掉结果中的重复项
  • ORDER BY 可以将筛选后的结果排序
EXISTS/NOT EXISTS 用法
SELECT column_name
FROM column_name
WHERE EXISTS(SELECT column_name
FROM table_name
WHERE condition)
  • 可以类比聚合函数，EXISTS函数的返回值是TRUE/FALSE,即判断的是子查询语句是否有符合条件的数据，即子查询语句返回的是不是一个空表。
  • EXISTS可以满足多表查询时检验分别在两张表的两个属性是否存在相等的情况，配合子查询灵活练习使用
Union 用法
(SELECT column_name
FROM column_name
WHERE condition)
UNION
(SELECT column_name
FROM column_name
WHERE condition)
  • 作用是将多张表的数据合成到一张表中打出
  • 注意细节！Union的作用是将两张表中属性的值（tuple）合成，并不会在新的表中把属性合成改变列的数量。
    Union例子
Update 用法
UPDATE table_name
SET column_name1 = 'Manager',
    column_name2 = 18000
(WHERE condition);
  • 用于修改表中某一属性的数据
  • 如果不添加条件限制，更新对象就是表中对应属性的所有数据。
  • 新的值必须符合该属性的数据类型
DELETE 用法
DELETE FROM table_name
(WHERE condition);
  • 用于删除表中某一属性的数据
  • 如果不添加条件限制，将会删除表中所有行对应该属性的数据。
