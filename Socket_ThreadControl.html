服务器端 - Server类

private Socket _serverSocket;
private List<Client> _clientList = new List<Client>();
private IPEndPoint _ipEndPoint;

public Server(string ipStr, int port)
{
    SetIpAndPort(ipStr,port);
}

public void SetIpAndPort(string ipStr, int port)
{
    _ipEndPoint = new IPEndPoint(IPAddress.Parse(ipStr), port);
}
  • 声明Socket类服务器对象，用于将其与客户端绑定
  • 利用_clientList将所有接入的客户端存储起来，方便管理。
  • 这里IPEndPoint类指的是网络连接终端，不管是服务器还是客户端都要做这样一个声明，其由ip地址和端口号组成。
public void Start()
{
      _serverSocket = new Socket(AddressFamily.InterNetwork,SocketType.Stream,ProtocolType.Tcp);
      _serverSocket.Bind(_ipEndPoint);              //绑定IP和端口号
      _serverSocket.Listen(0);                      //开始监听端口号，队列里最多有10个同时处理
      _serverSocket.BeginAccept(AcceptCallBack, _serverSocket);
}
  • 实例化Socket类，这里使用Tcp协议，所以SocketType选择稳定的Stream模式，通道模式有利于建立长期连接。
  • 我们将声明的服务器终端地址和服务器绑定。
  • Socket类的Listen函数是一个队列，这里传入的参数不是对连接数量上限做限制，而是对连接过程中的或其他状态的挂起数量上限做限制，也就是排队的数量。这里我将其调为零，选择放弃listen机制控制连接状态，通过BeginAccept函数的回调机制在控制线程的同时，控制连接顺序
    [socket中的listen()参数]https://blog.csdn.net/HFZeng/article/details/93843442
  • 然后我们要将服务器调成处于可接受客户端连接的状态。这里有两个函数可以选择，分别是Accept()函数和BeginAccept()函数，这里声明一个规律，很多函数涉及到Invoke类型的功能时会涉及到占用主线程的问题，这时一般都会选择Begin开头的同功能函数开启异步处理，防止占用主线程，这也是两个函数的主要区别。
private void AcceptCallBack(IAsyncResult ar)
{
      Socket clientSocket = _serverSocket.EndAccept(ar);
      Client cli = new Client(clientSocket,this);
      cli.Start();
      _clientList.Add(cli);
      _serverSocket.BeginAccept(AcceptCallBack, _serverSocket);
}
  • BeginAccept函数需要传入回调函数，这里利用回调函数结尾再次开启一个新线程的模式达到了解决线程冲突的目的，也保证了对每个接入的客户端做管控的目的，使整个流程足够稳定，且代码简洁。
  • IAsyncResult类是BeginAccept函数开启异步委托后的返回值，ar即是传入的客户端参数，我们需要进一步在回调函数中将其与服务器SOcket绑定。
    IAsyncResult异步委托
  • Client类是我声明的客户端类，表示实例一个新的客户端对象。
public void RemoveClient(Client client)
{
     lock (_clientList)
     {
          _clientList.Remove(client);
     }
}
  • lock函数的作用是确保当一个线程位于代码的临界区时，另一个线程不进入临界区。如果其他线程试图进入锁定的代码，则它将一直等待（即被阻止），直到该对象被释放。有效地保证了不会误删掉某个客户端线程。

客户端 - ClientManager类

public override void OnInit()
{
    base.OnInit();
    clientSocket = new Socket(AddressFamily.InterNetwork, SocketType.Stream, ProtocolType.Tcp);
    try
    {
        clientSocket.Connect(IP, PORT);
        Start();
    }
    catch (Exception e)
    {
        Debug.LogWarning("无法连接到服务器端，请检查您的网络！！" + e);
    }
}
  • 和服务器端方法相同，新实例化一个TCP对象。
  • 这里的IP和PORT是目标服务器的IP地址和端口。
  • 一定要做好异常检测和防空检测，一定一定！！

服务器端与客户端的消息接收

public void Start()
{
_clientSocket.BeginReceive(msg.Data, msg.StartIndex, msg.RemainSize, SocketFlags.None, ReceiveCallBack, null);
}
private void ReceiveCallBack(IAsyncResult ar)
{
    try
    {
        int count = _clientSocket.EndReceive(ar); // 他接受了多少字节的数据
        if (count == 0) // 正常关闭
        {
            Close();
            return;
        }
        msg.ReadMessage(count,OnProcessMessage);
        Start();
    }
    catch (Exception e)  // 非正常关闭
    {
        Console.WriteLine(e);
        Close();
    }
}
private void Close()
{
    ConnHelper.CloseConnection(_mysqlConn);
    if (_clientSocket != null){  _clientSocket.Close();}
    _server.RemoveClient(this);
}
  • BeginReceive方法是一个异步方法，会开启一个新线程接收消息，为了防止消息接收冲突，我们采用与服务器接收客户端连接相同的思想，在接受一个消息的同时开启一个新的线程监听是否有新的消息。
  • 这里涉及到一个服务器关闭的方法，我们设立一个count去捕捉接收到的消息的字符数，正常消息无论如何是不会发送字符数为零的消息的，如果服务器捕捉到这种消息则可以当作客户端关闭进程的信号，主动关闭掉与其的连接。
  • 这里的msg是我实例化的一个消息类对象，专门用于消息的拆解，分发以及处理粘包分包情况的发生，会在另一篇博客中提及。
  • 在关闭客户端时不要忘记先断开其与数据库的连接，不然你懂得。
